Conversion rate

Nov 26 2019 Interviews 7 minutes read (About 1009 words)

The goal of this challenge is to build a model that predicts conversion rate and, based on the model, come up with ideas to improve revenue.

Challenge Description

We have data about users who hit our site: whether they converted or not as well as some of their characteristics such as their country, the marketing channel, their age, whether they are repeat users and the number of pages visited during that session.

The goal is to

  • Predict conversion rate
  • Come up with recommendations for the product team and the marketing team to improve conversion rate

What I have learned

  • data preprocessing
    • categorical values turn into numerical values (One-Hot-Encoding)
    • numerical value distribution, including response variable!! if some kind of imbalanced, then try other metrics like precision and recall.
  • Detailed suggestion

From the model, we need to convert source and country into numerical values. Here we use One-Hot-Encoding.

Imbalanced class

Here we can see that target value is very imbalanced, which reminds us should not use plain “accuracy” as the metric to judge a classifier is good or not.

Since both means are similar to the whole mean, we can say that the splitting process doesn’t import any bias.

Classify via Logistic Regression

If we use the default cutoff 0.5, we can see a high precision (0.85) and a not very good recall 0.68, which means that we classify 32% converted user as non-converted.

Since our task is giving Sales team and Marketing team some insights, so we don’t need to tune the cutoff value now, since that won’t affect the model. However, if, for example, if we care more about not losing a valued customer, so incorrectly classifying a converted user as non-converted will pay much, then we need to lower down the cut-off probability to be 1, which will increase the recall with the price lower than precision.

Conclusions and Suggestions

  • Conversion among China users are very bad. We can see that there are lots of users coming from China, but their conversion rate is very low. Considering the huge market in China, we should try to improve conversion rate in China. We can consider whether our website has good Chinese translation and satisfy Chinese users’ culture requirement.
  • Conversion from German is great. But there aren’t lots of users from German. We can try some stratgies to attract German users like more advertisements.
  • Old users have better conversion rate. So we should keep in touch with them, e.g. send promotion email or coupons to old users.
  • Age is a negative factor. the older the user is, the less likely he/she will convert. Then we should find the reason behind it.
  • Total page visited is a positive factor. the longer the user browse our web page, the more likely he/she will convert and buy something. Then if we find some user browse a lot and didn’t buy anything, we can send email to remind them.
